A dev environment for my API Hub

The backend code of the API Hub is completely Dockerized. Luckily Microsoft Visual Studio Code has an excellent extension for Docker and Docker Compose: What is even better is that the integration between editor and codebase can still be taken one step further. With the Remote - Containers extension installed it is possible to run Visual Studio Code inside a Docker container.

Just open the API Hub repository as cloned in the previous blog post in Visual Studio Code (it goes without saying that the above mentioned extensions must be installed): cd ~/Documents/dev/api_hub_rpr_v4 code . and then click the reopen in container button: The workspace created is running inside a container. Execute, in a terminal window (keyboard shortcut ctrl + shift + `), the following commands to start the API Hub's webservices: cd backend/node/src npm install npm run dev

Testing the endpoints exposed by the API Hub will be the topic of the next blog post.
