Multilingual company names

In my previous implementation of a user interface for D&B Direct+ Data Blocks I had made a, what at the time was, fairly complete application for presenting the content of collections of data blocks to non-technical users. However, over a period of twelve months, a lot of data-elements were added to the API responses. The inclusion of arrays of multilingual company names definitely is one of the most interesting ones though. Moreover, what is great is that these often logosyllabic names are, if relevant, included immediately at the lowest level of the Company Information data block.The seamlessness of the implementation of these names is truly amazing for someone who has struggled with Windows code pages in the past.UTF-8 can basically handle any language you throw at it and is, in a completely unassuming way, the default character set in my entire development toolchain!

I had to make one major implementation decision and that was whether to, if available, implement both the regular and multi-lingual name versions or just choose one of the two. As can be seen from the code commit and the screenshot below I opted to, if available, display the multi-lingual names solely and, in those cases where no multi-lingual names are available, revert to the regular name property.
