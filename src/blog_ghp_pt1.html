Why migrate my blog to GitHub pages?

Previously I hosted my blog on hdr.is-a-geek.com. The following technology stack powered this site;

  • Intel NUC for hardware
  • Ubuntu as operating system
  • Apache as Web server (incl SVN for version control)
  • XHTML as markup language
  • JavaScript for client-side code (incl. jQuery)

My Intel NUC proved to be a trooper 💪

When I learned about GitHub Pages I just knew I had to port my blog over to this delivery platform. Main reasons for making the switch were;

  1. Unrivalled simplicity while, at the same time, meeting the service levels associated with professional hosting providers (please note that the uptime of my amateur stack wasn't half bad),
  2. A completely serverless experience with;
    • no more worries about catastrophic SSD or Intel NUC failure and
    • no more C code 🤓,
  3. Higher SEO scores because the domain github.io is more reputable than is-a-geek.com (even though my reputation isn't too shabby),
  4. An excellent opportunity to;
    • migrate from XHTML to HTML5 (semantic elements already implemented, a responsive webdesign noted as a to-do) and
    • switch from SVN version control to Git(Hub) and
    • loose the jQuery code in my client-side scripts.

After porting nearly all of the functionality of my blog & more than 20 blog posts, I think I can safely say that the investment of my time thus far has been well worth it and the result speaks for itself!
