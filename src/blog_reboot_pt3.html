Blog reboot, part III

My new GitHub repository is actually the 3rd version of my API hub. In hindsight I consider the 1st version of the codebase almost as a set of prototypes. At the time the re-use of code didn't feature very high on my agenda. Proving conceptual viability was the name of the game. The 2nd version of the project was all about Dockerizing the application platform. I therefore made a conscious decision to focus on just one API (i.c. D&B Direct+). At this stage my aim is to proceed with multiple API's but simultaneously I want to make sure that I effectively re-use the code I develop.

When it comes to effective code re-use the video below truly was an eye-opener for me.

From the days I learned C++ I've always been very much focused on inheritance but in the video MPJ makes a very strong case to use composition instead of inheritance. Of course I was not able to immediately incorporate all the ideas put forward into my GitHub repository but I succeeded in moving a lot of function code out of classes so as to share it between classes. As a result the current version of the codebase contains just one instance of method invocation https.request compared to six previously. Everything now is neatly parameterized & encapsulated in function execHttpReqResp.

Please note that function execHttpReqResp is invoked using the call method as defined on JavaScript's Function constructor because, in the body of the function, object instance properties are dereferenced using "this".

In my next blog I will delve a bit deeper into "this" but I highly recommend MPJ's video on "bind and this" as well.
