Country selection user experience

This blog post is all about one of my pet peeves, the sometimes awful user experience, on the Web or in desktop applications, when having to specify a particular country. I dislike the long dropdown with only limited ways to make use of the keyboard and text boxes which provide no input clues whatsoever. After reading the excellent article "Redesigning The Country Selector" I knew that I was not alone though.

For me the greatest challenge while trying to find a good country selector plug-in was that most components introduce a dependency to either a library or a framework. For example the autocomplete suggested in the aforementioned article depends on jQuery. The same is true for this very popular typeahead. I even considered a UI widget developed by the UK government but I discovered it has a dependency on Preact. In essence there is nothing wrong with either jQuery or Preact but, at this stage, I do not want to introduce these kinds of dependencies.

For this reason I was very happy when I found Pixabay's autoComplete. The title of the widget's demo page promises an " ... extremely lightweight & powerful vanilla JavaScript completion suggester ... " which is exactly what I was looking for. I must say that, with a last commit on Github three years ago, development of the selector code is not particularly active but still good enough for me.

The project's demo page contains an example country selector but, in my humble opinion, the code appears to be a bit dated with all kinds of loops and no arrow functions. Furthermore, a database of country flag images is referenced which adds nice eye candy but for me is just too much of a hassle to implement.
