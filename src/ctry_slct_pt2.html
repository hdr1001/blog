International IDentity Resolution

This weekend I revisited my blog posts (I, II, III, IV & V) about the vanilla JavaScript D&B IDentity Resolution user interface I developed one and half years ago. In part IV of this series I list the functionality an IDR UI should support and state that " ... all this functionality is now available". From a high level perspective this is a true statement but I can vividly remember that, at the time, I was desperately looking for a re-usable vanilla JavaScript type-ahead component for specifying a country code. My inability to find such a component convinced me that it was time to start learning a JavaScript UI framework. I took on a couple of other projects before, in April 2020, I started on my journey to learn React.

By now I have converted my entire vanilla JavaScript codebase to React. One of the benefits of switching to React is that comes with a huge collection of reusable, open source licensed components. At this stage I am not ready to commit to one of the larger frameworks like Material-UI or Bootstrap but it is possible to find excellent individual components as well. I chose Misha Moroshko's Autosuggest as a starting point for the purpose of developing a control which helps a user select a valid ISO 3166-1 alpha-2 country code. I already posted on LinkedIn about my CodeSandbox.io adventures with this control.

I'm happy to report that I have concluded the prototyping phase and have successfully incorporated the control in my API Hub UI code.

I took away two lessons while integrating the prototype code in my API Hub UI; (1) if you wonder whether you should lift state you probably should and (2) moving everything related to the country data to a separate file really cleaned up my code.
