Setting up my dev environment, Postman

I am in the business of using and building APIs. It should therefore come as no surprise that Postman occupies a pretty prominent place in my toolbox. I am aware of the fact though that alternatives do exist:

  • Like so many I started testing APIs using SoapUI. SoapUI is still around and still is a great tool but, as its name suggests, its focus for a long time solely was on SOAP and the application basically contained no facilities for testing REST APIs. For me that opened the door to Postman.
  • Lately I've seen a lot of hype surrounding Thunder Client. Nice thing about this Visual Studio Code extension is that it is build right into my favourite code editor. However I doubt whether this tool will cover all my needs. For now I have decided against making an immediate switch but I am definitely keeping an eye on the further development of this API client.

Postman excels, in my opinion, in the area of sharing predefined collections of API calls. I usually just right-click the relevant three dot menu and choose "Share collection": I then share using a JSON link: Please be aware though that anyone who has the link can access the collection and, if you are not careful, secrets (for instance credentials) can end up publicly available on the Internet.

Installing Postman is easy, just download the 64-bit Linux file from the Postman website and execute: sudo tar -xvzf ~/Downloads/Postman-linux-x86_64-X.X.X.tar.gz -C /optFor more information go to the Installing and updating chapter on Postman's online learning center. It contains a dedicated section on Linux.
