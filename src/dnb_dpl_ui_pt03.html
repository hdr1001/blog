Reading a D&B Data Block product

Yesterday I added a file selector control to my project. Today I wrote the code to actually read the file specified by the user. In this code the fileReader component plays an important part.

I would qualify the code above as classic JavaScript, (1) instantiate an object, (2) execute an async method on the object and (3) handle the method results in an event handler function. The code is constructed pretty defensively with a try-catch block guarding against parse errors and a check for the organization property which is common to all Data Blocks. The DUNS and the primary name of the entity in question are displayed in an alert box when the Data Block JSON is successfully read & parsed.

Agglo-Bouw is a completely fictitious company.
