1st run, pt. I

I have made the JavaScript for my D&B Direct+ prototype part of my Subversion cheat sheets repository. The shortest route to the code is therefore, in language familiar to people working with revision control systems, to just check it out. For this you will need Subversion: apt-get install subversion

With Subversion installed I was able to check out the code. I didn't want to do this as root though. I want to run the code as user dnb_dpl and for that reason, as root, I performed a simulated initial login of user dnb_dpl: sudo -i -u dnb_dpl

Next I created a directory dev in the home directory of user dnb_dpl and checked out my cheat sheets to subdirectory cheat.

I now was able to take a peak in the files, for example: vim ~/dev/cheat/node.js/dnb_dpl_objs.js
