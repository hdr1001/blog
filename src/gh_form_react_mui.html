A React Material UI form hosted on GitHub

Last year I decided I would no longer develop web user interfaces using vanilla JavaScript. Instead I invested in learning React. As a first practical exercise I ported my D&B Direct+ IDentity Resolution vanilla JavaScript user interface to React. Even though, from a purely functional perspective, the front-end worked very well, I thought the design was a bit bland. Of course I could have, at least partially, fixed this issue by investing a lot of time in the project's style sheets. Unfortunately web design is not one of my primary interests. I therefore decided to start using the Material UI library. Material UI (aka MUI) provides a robust, customizable and accessible library of foundational & advanced React components. These components all adhere to Google's Material Design guidelines and therefore, in my modest opinion, just look good.

Before developing a significant project I thought it would be a good idea to test MUI in a smaller, proof of concept like setting. I opted to make a new version of the form that I have used as a template in multiple builds now. For comparison the previous version of the form is hosted on GitHub as well.

Comparing the two final products I noticed how different the dimensions of the forms are. The React form is vertically more compact despite the fact that is has a title and vertically organized radio & option buttons. The React MUI form has more horizontal space, especially for the text input fields.
