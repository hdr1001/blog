Creating an Ubuntu 18.04 VM

Immediately upon conclusion of installing the Ubuntu Bionic Beaver desktop environment I wanted to proceed and install the server edition of this OS as a virtual machine on my T430. In fact this is a two-step process: (1) installing the virtualization software (including the VM manager user interface) and (2) implementing the server edition of Ubuntu 18.04 as a virtual machine. For the first step one of my aims was to validate whether this article on How-to Geek is still valid when it comes to installing KVM in the context of the latest release of my favourite Linux distribution. Although I'm quite sure it is, I can't confirm this with certainty because, in first instance, I made a bit of a mess of the installation. The reason for me botching a relatively straightforward task was that I forgot to enable the CPU's virtualization features in BIOS. A rookie mistake and, even worse, a mistake I've made before. Anyway, when installing virtualization on Ubuntu, my advice is to always start with checking whether this feature is enabled in BIOS. In my experience it's disabled by default. Once enabled you should be good to go if you follow the instructions on How-to Geek.

As for the second part of the process, implementing the server edition of Ubuntu Bionic Beaver as a virtual machine, I've documented the steps I took to accomplish this part of the project below.

Step 1: Clicked the "Create a new VM" icon in VM manager

Step 2: Installed the OS using local media

Step 3: Specified the path to the ISO & OS version (18.04 not yet available!)

Step 4: Chose memory & CPU settings

Step 5: Created a 10GB VM disk image

Step 6: Clicked finish (name should of course have been vm1804)

Step 7: Server install, installation procedure language

Step 8: Ubuntu Bionic Beaver server install

Step 9: Selected English as the preferred language for my server

Step 10: Chose UK (ext. with Windows keys) keyboard layout

Step 11: Confirmed (traditional) Ubuntu server install

Step 12: Went with default network interface configuration

Step 13: For me there was no need to specify proxy details

Step 14: Set up the filesystem to use the entire disk

Step 15: Installed to virtual device /dev/vda

Step 16: Confirmed destructive action (sounds ominous but is okay)

Step 17: Filled out profile details with server name vm1804

Step 18: Upon completion the finished install log was displayed

Step 19: Ubuntu server 18.04 welcome screen after reboot

Please note that I was annoyed that, in a brief moment of inattention, I had chosen the wrong name for the virtual machine. Unfortunately this is not a simple label that can easily be changed. However, I was sufficiently annoyed to clone the VM in order to give it its proper name: vm1804.
