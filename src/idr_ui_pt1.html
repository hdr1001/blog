IDR UI, 1st attempt (100% HTML)

In my previous three blog posts I (1) wrapped D&B's Direct+ IDentity Resolution functionality in a JavaScript class, (2) added persistence to this class & (3) connected the object logic to custom web service endpoints. While developing this code I did all my testing in a Firefox add-on called RESTClient. Of course, in real life, IDentity Resolution functionality is often used by end users to lookup a company on the D&B database. In these scenarios a User Interface is needed. The development of such a UI, in HTML, will be the subject of my next series of blog posts.

My plan for the first implementation of the user interface was simple; recycle some of the cheat sheet code I developed a couple of years ago and, from there, just stick to the age-old adagium "keep it simple, stupid". In practice this last bit meant no client-side JavaScript code (at least not in v0.1).

Effectively I took the following steps to get to a working UI:

  1. I checked out my tableless XHTML form from the publicly available cheat sheet repository and incorporated it into my project. Of course I had to rework the code. The most important changes I made were (1) I deleted (or hid) a bunch of input elements, (2) I changed the values associated with the for, name & id attributes of the label & input elements to reflect the query string parameters of the Direct+ IDR call and (3) I updated the form tag's method & action attributes. Please note that, in this first version, the country code parameter is fixed to NL in a hidden input field.
  2. Besides the tableless XHTML form I also made the accompanying cascading style sheet part of my project. I just copied the file, renamed it to dnb_frm_idr.css and, using a link tag, created a reference to it in the header section of the test page.
