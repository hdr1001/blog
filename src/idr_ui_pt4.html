IDR UI, pick a match candidate

For me this blog post represents another important milestone. IDentity Resolution is a big topic and when I embarked on my journey my goal was to take care of the IDR back-end stuff first (see here, here & here) and then build a fully functioning front-end. In my opinion such a front-end must support the following functionality:

  1. user specification of the search criteria,
  2. user submission of the specified criteria,
  3. processing of the submitted IDR request,
  4. feeding the generated match candidates back to the end-user,
  5. enabling the user to pick a candidate from the list and
  6. submission of the chosen candidate for the purpose of persistence.

I'm happy to report that all this functionality is now available on my SVN code repository.

My previous post contains a screenshot of the user interface, as it was a week ago or so, displaying a list of match candidates. This same collapsible element now looks like this:

Obviously there are multiple changes but the biggest difference no doubt is the inclusion of radio buttons on the form:

The radio buttons allow the user to choose a candidate and then the DUNS associated with this particular candidate can be submitted for storage on the back-end server. Clicking the Submit button will also revert the application to an empty search criteria form allowing the user to specify a new IDentity Resolution request if needed. In order to test the application I decided to look-up the top 5 teams of the second division football league in my home country (i.e. countryISOAlpha2Code=NL).
