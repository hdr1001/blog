IDR UI, evaluating match candidates

In my previous post I explained the reasons why, in my opinion, my JavaScript user interface code for D&B identity resolution had reached such a maturity level that it could be considered a fully functioning front-end. Of course, in this case, "fully functioning" by no means implies "best-in-class". In order to take the implementation further in that direction I will, in this post, add a couple of lines of code to help the end-user pick the right match candidate. In order to achieve this I leverage additional match candidate attributes as returned by the API and make them an integral part of the UI. Please refer to this excellent guide on the D&B Direct+ website for more information about these additional attributes.
