D&B IDR JavaScript object code

Nearly 6 months ago I published a blog post about a Node.js web service which was supposed to be the starting point of my implementation of D&B’s Direct+ IDentity Resolution (IDR) functionality. I followed-up with a couple of posts in which I used IDR as an example to illustrate basic JavaScript/Node.js concepts like asynchronous programming, arrow functions, promises and error handling. In the process I developed a function for returning the name of the first match candidate as generated by D&B. At that point I was, unfortunately, distracted by a couple of questions which happened to cross my path but now I’m back on track and I have made the IDR code part of my foundational Node.js Direct+ object code.

At the time of writing this post, August 2018, the implementation of class DplIDR is still rudimentary. It basically encapsulates the functionality exposed by the D&B Direct+ IDR endpoint. I know this is not really earth-shattering (as a matter of fact I did the same thing here) but I love being able to code this:let oIDR = dpl.getIDR({name:'Agglo', countryISOAlpha2Code:"NL"});The instruction above instantiates an object of class DplIDR and, upon load, the object instance will contain data related to match candidates which fit the following search criteria: company name like "Agglo" and located in "NL" (i.e. The Netherlands).
