D&B IDentity Resolution candidates

In my previous post I explained the form logic of my D&B Direct+ IDentity Resolution application without listing any match candidates. I'm happy to report that this functionality is now available in my code as well. Currently the UI looks like this in a test environment;

I tried to make the form as easy to work with as possible. To accomplish this my aim was to (1) make the specification & submission of the search criteria frictionless and (2) to present the available match candidate information in a simple but comprehensive way. I therefore limited the number of match candidates listed to five and used, for trade-style names, the mail address and the CEO name, icons which, when hovered, display the relevant information.

For this last functionality I use React's conditional rendering capabilities;

In order to render multiple match candidates I also use React's concept of lists & keys;

This means that I have now implemented all of React's main concepts in my API HUB user interface code (for a listing of the other examples click here). Moreover, I have already used more advanced concepts like references and useState as well. I guess I can say I'm getting the hang of React and I must say I like it!
