Move the prototype code

I've been hosting my cheat sheets in a public SVN repository for years now. When I started work on my D&B Direct+ prototype I took the easy route and just added a directory node.js to this repository. But now my plan is to expand the existing code base and add an entirely new Apache module. The time has therefore come to create a dedicated SVN repository for hosting my D&B Direct+ prototype.

Please note that all the necessary software (SVN, Apache, ...) to complete the steps below was already available on my server! To add the existing Node.js code to the newly created repository I checked out revision 0 of the code base (which of course was completely empty) to my Ubuntu virtual server. I then created a subdirectory node.js and copied the relevant files to this location. Everything was ready for a first commit once I had put the JavaScript files and the license file under version control while, at the same time, explicitly ignoring other items.

I decided not to remove the Node.js code from my cheat repository but to warn users that the code is now maintained elsewhere.
