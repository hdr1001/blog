Constructing the platform, pt. II

In part I of this series an Ubuntu powered Digital Ocean Droplet was created. With this foundation in place installing Apache is easy. As super user (alternatively use sudo) I just executed: apt-get install apache2 This command will download and install Apache on the cloud server. Of course the end result will be entirely based on Ubuntu defaults. By and large these defaults work just fine. The only configuration I usually do at this stage is to set the ServerName directive. This can be accomplished by first opening the main Apache configuration file: vim /etc/apache2/apache2.conf and then adding the following directive to the end of the file: ServerName specify_a_name After executing these steps I was able to make a client request for the Apache default page.

Please be aware that, for me, the steps above were just a repeat of the implementation work I had done a couple months earlier while developing the D&B Direct+ prototype. My main goal was to make sure that I was accurate in my description of the essentials of constructing the platform. I therefore worked under the assumption that, at some point, I would pull the plug on the Droplet and, as a result, non-essentials had no priority. Below just a quick mention of two examples of what, for the purposes of this blog, I considered to be non-essentials but which, for some readers, might be good tips nevertheless:

  • In the screenshot above you can see that the Apache default page was requested based on the server's IP address. Of course it's preferable to link a web site to a domain name. For this purpose I normally use Dyn DNS. This company (dynamically) manages the connections between (1) the Intel NUC situated immediately besides my home fuse box & hdr.is-a-geek.com and (2) my more permanent Digital Ocean Droplet & hdr.office-on-the.net.
  • Speaking of more permanent installs, it's quite natural that, at some point, the need arises to get more insight into visitor statistics. GoAccess is my favourite tool to gather and visualise this kind of data. It's an extremely lightweight tool but it parses the information contained in the Apache log files at the speed of light and presents the relevant statistics in an orderly fashion.
