Getting started with React

My API Hub is a classic example of a server side solution. The key based requesting, persisting and passing on of data products requires no user interface. The process of establishing a product key is different though. This usually involves the specification & submission of search criteria and the evaluation of the match candidates returned. A high quality user interface can make a big difference when trying to locate a key.

I am still 100% committed to creating an outstanding UI for locating keys but, whereas previously my goal was to create such a UI based on a combination of HTML/JavaScript & CSS code, I now want to use React for this purpose. I could go into detail to try and explain the reasons for changing my approach but why should I invest my time if I can just refer to Chris Achard's excellent article "React vs. Plain JavaScript". The only thing I have to add are a JsFiddle and a Code Sandbox based on the JavaScript fragments listed in the article.

If you want to learn React like me, a quick Google search will return lots & lots of freely available tutorials on the Internet. After a bit of consideration I decided to use React's own guide to (the) main concepts as my primary educational resource. I am the kind of person who prefers to learn step-by-step and, for me, this tutorial proved to be a perfect fit. Of course I like to experiment as well. Out of the box the guide comes with CodePen examples so as to make the learning experience more interactive.
