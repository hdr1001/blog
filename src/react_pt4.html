Furthermore, in case you are a regular reader of this blog, it should come as no surprise that I am very much interested in JavaScript playgrounds.

Considering the above it is only logical that, as a next step, my aim was to put together a local, Ubuntu based, IDE to develop React applications. It is important to note that, while working on this project, I kept the following two objectives in mind;

  1. I wanted to stay as close as possible to the online CodeSandbox IDE and
  2. I did not want to use Create React App. Even though setting up all the build tools, dependencies & configuration files by running a single command sounds tempting, it will not help with the creation of the (primarily) educational experience I am looking for.

I kicked off the project by initiating a new GitHub repository online and cloning it to my PC; cd ~/Documents/dev git clone https://github.com/hdr1001/min_react_app

Of course the Node Package Manager is an important part of the entire setup. I installed it locally with this command; curl https://www.npmjs.com/install.sh | sudo sh

VS Code was already available on my machine with the extensions listed above

To continue with my work I just opened the project's main folder and followed the instructions in this YouTube video;

Nearly all the instructions in the video worked perfectly for me with one exception, in order to install Babel I used; npm i --save-dev @babel/core babel-loader @babel/preset-env @babel/preset-react html-webpack-plugin The associated Babel presets in the file .babelrc are different as well; {"presets": ["@babel/preset-env", "@babel/preset-react"]} I also added an additional CSS loader to my project; npm install --save-dev css-loader npm install --save-dev style-loader To make the style sheets part of the project's build I added the JavaScript object below to the rules array of file webpack.config.js; , { test: /\.css$/i, use: ['style-loader', 'css-loader'] }

At this point I was ready to test the, admittedly minimal, application. npm start Everything worked just fine on the 1st try. Building the final product went smoothly as well. npm run build

I have committed & pushed my changes to GitHub. On another PC I could clone the repository, install the necessary packages using ... npm install ... immediately followed by ... npm start ... to start the application. üëç
