I ❤ Ubuntu

I can trace my history with Linux all the way back to the early days of Slackware. Just out of interest I would install this distro on outdated hardware and experiment. For a short time I switched to Suse but, at the time, my impression was that this OS was a bit of a resource hog. That's why I gave Ubuntu a try. Ever since I have used Ubuntu on both my desktop and my servers. The server version of this OS powers my Digital Ocean droplets in the cloud and is installed on the Intel NUC which I privately own and use to host this blog. Nowadays I tend to stick to the long-term support releases. The software I employ is mature and therefore there is no real need to update more frequently than once every two years.

For me Ubuntu is a platform on which I can develop my projects and prototypes. The software I use to build on this foundation (e.g. Apache, PostgreSQL, Node.js, ...) can be found in the standard repositories of the OS and therefore a simple: sudo apt-get install (apache2, postgresql, nodejs, ...) suffices to retrieve, install and implement the server components. Minimal configuration is usually needed but my aim is always to stick, as close as possible, to the configuration as provided out-of-the-box. From a support perspective it's reassuring to know that, for LTS releases, Ubuntu guarantees the availability of security patches & updates for a period of five years.

I briefly want to mention that connecting to Ubuntu server instances over SSH is something I do on a daily basis. Remembering the relevant user-IDs and passwords becomes more and more challenging as the number of servers that you have to support grows. Reusing passwords is tempting but always a bad idea. I therefore have switched to using SSH public-private key pairs on all my Linux boxes (this includes my Raspberry PI). In practice this is not only very convenient for authentication purposes but also improves security. Digital Ocean supports SSH keys as a standard feature of their droplets and has produced a couple of good community articles on the subject.
