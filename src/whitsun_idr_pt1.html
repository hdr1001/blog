My first action to get going with the development of a React based user interface for D&B Direct+ IDentity Resolution functionality was the creation of an entirely new GitHub Repository.

I wanted my minimal React app code to be the starting point for the new project but, at the same time, I did not want a permanent link between the two repositories, so I searched for alternatives to forking. Luckily I came across an article about "mirroring a repository" which turned out to be just what I was looking for.

Next I cloned the repository ... git clone https://github.com/hdr1001/api_hub_rpr_ui.git ... and made some changes to reflect the fact that the purpose of the code had changed.

In order to run the code I executed ... npm install ... from the project's root directory. There were some vulnerabilities but those were easily fixed with ... npm audit fix ... and the command above prepared the way for ... npm start
