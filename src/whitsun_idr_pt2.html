In my previous blog post I created a new React, Webpack & Babel based repository which will contain my user interface component for D&B's Direct+ IDentity Resolution API. The gist of D&B IDR is simple, as a user you can submit search criteria to the cleanseMatch endpoint and, in response, the API will return match candidates which, in one way or another, match the criteria specified. Considering the application logic it is only natural, as a 1st step, to develop a simple form to enable a user to fill out search criteria & that is just what I did in this commit.

Currently there is no interaction with the API and clicking the Submit button will only list the criteria specified by the user in an alert;

I'm well aware that this is nothing fancy but I was struck by how many elementary React concepts are already built into the application even at this very early stage.
