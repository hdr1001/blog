IDR user interface application logic

I recently added a 2nd form and the accompanying application logic to my React based D&B Direct+ IDentity Resolution form. The IDR endpoint will basically generate match candidates for a set of search criteria submitted. Considering this functionality it makes sense to have two separate parts to a UI which will interact with the endpoint,

  1. a form which allows the end-user to specify search criteria (this was the subject of my previous post) and
  2. a form which allows the end-user to evaluate the match candidates and select the correct one.

Of course the order of the process is clear. The process is started by the specification & submission of the criteria, then there will be a brief wait state and, finally, the generated match candidates will be presented and the end-user should be able to select & confirm one. This last part will the subject of my next blog post. For now there is just the framework & the logic;

With this in place I can now focus on the actual API request & response (against my API HUB, not immediately against Direct+) and the React components needed to feedback the candidate information generated in the transaction. Please stay tuned for more ...
