Web service, pt. I

In the previous post I ran a simple example application from the Linux command line to test the objects defined in the file dnb_dpl_objs.js. This example application has a well-defined beginning & end and therefore, upon completion, it's possible to close the shell session used for hosting the application or use it for something else entirely. This is different in case of my Node.js web service implementation of my D&B Direct+ prototype. I've had this process up-n-running for weeks on end. Having an active shell session running for all this time is just not practical. Enter GNU screen. GNU screen makes it possible for programs to continue to run even when a whole screen session is detached from the user's terminal. Then, at a later time and maybe even from a different location, it's possible to re-attach to the screen session in order to, for example, view the debug messages logged by the application.
